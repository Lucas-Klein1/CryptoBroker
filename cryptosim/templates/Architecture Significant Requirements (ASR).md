# Quality Attribute Scenarios – CryptoBroker (Stand: 20.11.2025)

| **Qualitätsattribut (Quelle)** | **Stimulus** | **Artefakt / Gegenstand** | **Umgebung** | **Antwort** | **Antwortmaß (Messgröße)** |
|-------------------------------|--------------|----------------------------|--------------|-------------|-----------------------------|
| **Benutzerfreundlichkeit** | Ein neuer Nutzer öffnet erstmals das Web-Dashboard. | Benutzeroberfläche (UI), Navigationsstruktur | Normalbetrieb | Das System zeigt ein klares, intuitives UI mit geführter Erstnutzung (Onboarding). | Nutzer findet gewünschte Funktion innerhalb von **max. 5 Sekunden** / max. **2 Klicks**. |
| **Benutzerfreundlichkeit** | Ein Nutzer gibt fehlerhafte Eingaben in ein Formular ein. | Eingabevalidierung, UI-Formulare | Normalbetrieb | Das System zeigt verständliche, kontextbezogene Fehlermeldungen. | Fehlermeldung erscheint innerhalb von **< 1 Sekunde**, Fehlerbehebung ohne externe Hilfe möglich. |
| **Zuverlässigkeit** | Ein Server fällt aus. | Backend-Services, Server-Cluster | Überlast- oder Ausfallbetrieb | System führt automatisches Failover durch und nutzt redundante Instanzen. | Ausfallzeit < **5 Sekunden**, kein Datenverlust. |
| **Zuverlässigkeit** | Eine externe API (z. B. Preis-Feed) liefert temporär keine Daten. | Externe Service-Schnittstellen, Datenfetcher | Normalbetrieb | System wechselt auf sekundäre Datenquelle oder liefert gecachte Werte. | System bleibt funktionsfähig; max. **30 Sekunden** Datenverzögerung. |
| **Leistung (Performance)** | Ein Nutzer ruft die Startseite auf. | Frontend-Rendering, API-Endpunkte | Normalbetrieb | Schnellladeprozess inkl. Caching, asynchroner Datenabruf. | Seitenladezeit **≤ 2 Sekunden**. |
| **Leistung (Performance)** | 10.000 Nutzer greifen gleichzeitig auf Live-Kursdaten zu. | Datenstreaming-Service, Echtzeit-API | Hochlastbetrieb | Load-Balancing und horizontale Skalierung werden aktiviert. | Antwortzeit **< 300 ms**, keine Abbrüche. |
| **Sicherheit** | Ein nicht autorisierter Nutzer versucht, auf das Dashboard zuzugreifen. | Authentifizierungs- & Autorisierungsmodul | Normalbetrieb | System verweigert Zugriff, loggt den Versuch und informiert Sicherheitsmonitoring. | Zugriff blockiert in **< 100 ms**; Angriff im Log dokumentiert. |
| **Sicherheit** | Ein Nutzer startet eine Transaktion (Kauf/Verkauf). | Transaktionsmodul, Datenverschlüsselung | Normalbetrieb | Verschlüsselte Verarbeitung, Identitätsprüfung (2FA). | Transaktionsverarbeitung **< 2 Sekunden**, vollständige Verschlüsselung. |
| **Wartbarkeit** | Ein Entwickler deployt ein Update während des Betriebs. | Backend-Services, CI/CD-Pipeline | Normalbetrieb (Rolling Deployment) | System aktualisiert Module ohne Unterbrechung. | Deployment ohne Downtime; Rollback in **< 10 Sekunden**. |
| **Wartbarkeit** | Ein Fehler wird im Log entdeckt. | Monitoring-, Logging- und Diagnose-Tools | Normalbetrieb | Fehler kann reproduziert und isoliert werden. | Fehleranalyse dauert **< 15 Minuten**. |
